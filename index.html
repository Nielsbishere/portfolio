Wisp

Realtime raytracing


Description

Wisp is a real time raytracing library that can be integrated into your favorite engine or tool, made by a team of nine students. It is currently used in a Maya extension that allows you to view a deferred, full raytracing, hybrid and path traced preview of the current scene.
The goal of the project is to allow artists to have a quick overview of what their end-product would look like; saving time and increasing their workflow. Maya currently doesn't provide that in the regular viewport, so the deferred preview is a step up to view the effects of modeling real-time. Previewing the reflections, global illumination and shadows is a nice bonus for RTX and GTX Pascal cards.

Mineblowers

Custom controller game

Description

Mineblowers was a marmalade game made in 3 weeks, which utilises a custom controller. The idea was to create a unique user experience with an unexpected controller.
We chose for a bike pump as a controller, because it was easy to obtain and it fit in with our minecart-style game.

Osomi Core

Cross platform framework

Description

Osomi core is a cross platform framework, designed to be API and platform independent. It is created with Vulkan, Android and Windows in mind, so it has things like memory management (CPU & GPU), file management, input and serialization.
Other features include shader reflection, custom file types for models (to support sized shader inputs), shaders (to support reflection) and more. The architecture is created to be abstract so that the user doesn't have to think about platform/API specifics.

Harpoon

Unity C# Injector

Description

Harpoon was a project that was used in Shiploader; a modloader for the game Raft. This API was intended to be used by the modding community as an alternative. This was due to the existing modding tool using the source code of the game and distributing it and mods being very version dependent.
The alternative was code injection and not just patching a dll; where an executable requests Unity to load mods, which use a library for wrapping the low-level code.
This is accomplished by loading a C DLL into memory, which calls functions provided onto a heap; these functions are Unity mono functions, which load a C# dll and execute it. The C# dll is then able to access everything in Unity through C# reflection and Harmony through patching functions (More info here).
